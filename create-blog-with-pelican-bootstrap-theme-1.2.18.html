HOW TO CREATE A DATA SCIENCE - MACHINE LEARNING - DEEP LEARNING BLOG using Python, Jupyter Notebooks, Pelican, and Bootstrap

Date Tue 02 January 2018 By Brian Griner, PhD Category posts. Tags Python Data Science Pelican Blog

This post is targeted to Python programmers and data scientists who want an easy way to create a website or blog using Jupyter Notebooks or Markdown documents.

Enter Pelican. Designed to transform Markdown documents like Jupyter Notebooks into HTML that is ready to host on a web server like github.io via GitHub Pages.

So how do blogs produced by Pelican look? Okay but it could be better.

What follows is a complete 'recipe' for integrating a pelican bootstrap theme into your Pelican blog for a very professional looking site. Since there are many steps involved, the rest of this post will be in outline form to create a 'cheatsheet' for your reference.

For more background on using Pelican to create a data scinece blog, see this excellent post from dataquest.io

QUICKSTART YOUR BLOG USING MARKDOWN

A. INSTALL AND ACTIVATE A VIRTUALENV FOR PELICAN BLOG SITE

  • pip install virtualenv

  • virtualenv ~/virtualenvs/pelican

  • cd ~/virtualenvs/pelican

  • . bin/activate

B. NEXT INSTALL PELICAN & MARKDOWN IN THE VIRTUALENV

  • (pelican) $ pip install pelican

  • (pelican) $ pip install Markdown

C. NOW RUN PELICAN-QUICKSTART FROM YOUR PELICAN VIRTUALENV AND INSTALL REQUIREMENTS

  • (pelican) $ pelican-quickstart

  • (pelican) $ answer questions: defaults except: URL prefix: N, TIMEZONE: America/New_York

  • (pelican) $ pip install -r requirements.txt

D. CREATE .md FILE IN CONTENT WITH BLOG

E. START DEVELOPMENT WEB SERVER TO VIEW WEBSITE IN BROWSER

  • (pelican) $ make devserver

  • IN BROWSER OPEN localhost:8000

F. TO STOP LOCALHOST SERVER

  • (pelican) $ ./develop_server.sh stop

HOW TO CONVERT A Jupyter Notebook INTO A BLOG POST

G. INSTALL JUPYTER PLUGIN AS GIT SUBMODULE

  • (pelican) $ git init

  • (pelican) $ mkdir plugins

  • (pelican) $ git submodule add git://github.com/danielfrg/pelican-ipynb.git plugins/ipynb

H. MODIFY pelicanconf.py TO INCLUDE THE FOLLOWING LINES:

  • MARKUP = ('md', 'ipynb')

  • PLUGIN_PATHS = ['./plugins']

  • PLUGINS = ['ipynb.markup']

I. COPY .ipynb INTO content FOLDER CREATED BY pelican-quickstart

J. NOW CREATE A FILE WITH THE MARKDOWN META DATA FOR THE POST IN CONTENT DIR USING THE SAME NAME AS .ipynb BUT HAS THE EXTENTION .ipynb-meta

Title: MY NEW BLOG POST

Slug: new-post FILENAME

Date: 2017-12-22 17:30 DATE POSTED

Category: posts Tags: python 'data science' 'machine learning' 'deep learning'

author: Your Name

Summary: BRIEF DESCRIPTION OF CONTENT OF NEW POST

K. GENERATE HTML INTO OUTPUT DIR FROM MARKDOWN FILES IN CONTENT DIR

  • (pelican) $ cd .. (OR ~/virtualenv/pelican)

  • (pelican) $ pelican content TO GERNERATE HTML IN output folder

L. START PELICAN WEBSERVER AND REVIEW BLOG IN BROWSER:

  • (pelican) $ cd output

  • (pelican) $ python -m pelican.server

  • (pelican) $ localhost:8000 IN BROWSER TO VIEW POST

M. CREATE A STATIC HOME PAGE USING MARKDOWN

  • (pelican) $ mkdir content/pages (ALSO content/posts TO STORE POSTS)

  • USE OVERRIDE FEATURE IN MARKDOWN METADATA FILE:

Title: The Title of your Home Page

URL:

save_as: index.html

Thank you for visiting my blog ...

USE PELICAN-THEMES TO INSTALL BOOTSTRAP2 THEME

N. INSTALL PELICAN THEMES FROM GITHUB SITE

  • (pelican) $ git clone --recursive https://github.com/getpelican/pelican-themes ~/pelican-themes

O. RUN pelican-themes TO LIST INSTALLED THEMES, INSTALL NEW OR REMOVE EXISTING THEMES

  • (pelican) $ pelican-themes -v -l TO LIST INSTALLED THEMES

  • (pelican) $ pelican-themes --install

P. ADD THEME VARIABLE TO PELICANCONF.PY WITH THEME NAME

  • THEME = "bootstrap2"

Q. RUN PELICAN WITH -t bootstrap2

  • (pelican) $ pelican content -s pelicanconf.py -t bootstrap2

R. CHANGE TO OUTPUT DIR, START PELICAN WEBSERVER, AND VIEW SITE IN BROWSER

  • cd output

  • python -m pelican.server

  • OPEN localhost:8000 IN BROWSER

NOW ADD GITHUB PAGES AS REMOTE TO LOCAL REPO

S. STOP WEBSERVER IN LOCALHOST

  • deactivate (./develop_server.sh stop?)

T. COMMIT FILES TO GITHUB

  • git checkout -b dev CREATE AND SWITCH TO BRANCH dev

  • ghp-import output -b master TO IMPORT ALL FILES FROM output TO master

  • git push origin master TO PUSH CONTENT TO GITHUB

U. CHECK yourusername.github.io FOR YOUR NEW BLOG POST!
